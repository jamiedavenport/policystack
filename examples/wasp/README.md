# Wasp Example — privacy policy + cookie banner in a Wasp app

A minimal [Wasp](https://wasp.sh) (v0.24) app showing the full PolicyStack
story end to end: one `defineConfig()` in
[`src/policystack.ts`](src/policystack.ts) generates the privacy policy and
cookie policy pages **and** derives the consent runtime behind the cookie
banner. The only Wasp-specific wiring is the root component in
[`src/Root.tsx`](src/Root.tsx), registered via `client.rootComponent` in
[`main.wasp.ts`](main.wasp.ts).

Unlike the [TanStack example](../tanstack), this project is **not** part of
the pnpm workspace — Wasp drives its own `npm install`, so it consumes the
published `@policystack/*` packages, exactly like a real user's project.

## What it demonstrates

- **One `defineConfig()` as the single source of truth** — policies and
  consent categories (locked `essential`, opt-in `analytics`) from one file.
- **One provider for everything.** `Root.tsx` mounts `<PolicyStack>` from
  `@policystack/react/provider` around Wasp's `<Outlet />`.
- **A hand-rolled banner + preferences panel** built from
  `useConsent()`/`useCategory()` — plain CSS, matching the Wasp starter style.
- **`<ConsentGate>` + a `consent.has()` guard** gating a demo analytics cookie
  in [`src/MainPage.tsx`](src/MainPage.tsx).
- **`PrivacyPolicy` / `CookiePolicy` pages** rendered through the app's own
  slot components via the `components` prop (see
  [`src/components/policyComponents.tsx`](src/components/policyComponents.tsx)).
- **The `policyStack()` Vite plugin** running inside Wasp's own
  `vite.config.ts` next to the `wasp()` plugin — source scanning, validation,
  and the opt-in consent scanner in warn mode.

## Project structure

```
main.wasp.ts                      ← app spec: routes/pages + client.rootComponent
vite.config.ts                    ← wasp() + policyStack() plugins
src/policystack.ts                 ← defineConfig() — the single source of truth
src/policystack.gen.ts             ← generated by policyStack()'s source scan
src/Root.tsx                      ← <PolicyStack> provider + nav + banner mount
src/MainPage.tsx                  ← "/" — ConsentGate demo + gated cookie write
src/pages/PrivacyPolicyPage.tsx   ← "/privacy-policy"
src/pages/CookiePolicyPage.tsx    ← "/cookie-policy"
src/components/CookieBanner.tsx   ← first-visit banner (useConsent)
src/components/CookiePreferences.tsx ← per-category toggles (useCategory)
src/components/policyComponents.tsx  ← slot components the policies render through
```

## Running locally

Requires the [Wasp CLI](https://wasp.sh/docs/quick-start) (`npm i -g
@wasp.sh/wasp-cli@latest`):

```sh
wasp install   # npm-installs dependencies (first run only)
wasp start     # compiles and serves http://localhost:3000
```
